Hue Emulator.

The Hue Emulator is a Bridge Emulator for the Philips Hue - Personal Wireless System.
It allows developers who don't have access to a Philips Hue system (i.e. A Bridge + Hue Light Bulbs) to write hue apps.

Note It should be noted that this is a Bridge API Emulator, ** it is not a hardware Emulator** so response times (for example the number of light/group commands that can be sent per second) will differ to an actual bridge. For a true test a Hue Bridge should be used.

How to use

The emulator is written in Java so a Java Runtime environment is required.

The emulator is an executable .jar file, so can either double click or from the command line:-

java -jar HueEmulator.jar

Enter the port you want your emulator to run on and click the Start button (this starts the Emulator/HTTP Server). To check the Emulator is running use the below URL. The full bridge config should be returned, in JSON format:

http://localhost:8000/api/newdeveloper

The root of the github project contains the HueEmulator.jar executable, so building the source is not required.

You can use the emulator a variety of ways:

Clip

The emulator accepts RESTful web service calls (in JSON format) just as the Hue bridge obviously.

For more details on the valid Hue API calls consult the documentation at: http://developers.meethue.com/

Java/Android SDK

If you are using the Java SDK you can connect to the emulator using the following lines of code:

PHAccessPoint lastAccessPoint = new PHAccessPoint();
lastAccessPoint.setIpAddress("192.168.1.1:8000");   // Enter the IP Address and Port your Emulator is running on here.
lastAccessPoint.setUsername("newdeveloper");        // newdeveloper is loaded by the emulator and set on the WhiteList.
phHueSDK.connect(lastAccessPoint);

If you are testing on Android make sure your Android Wifi and your PC are on the same Network/Wifi. Additionally if connection failures occur your firewall settings may need adjusting.

See the code examples on the Philips GitHub page for more info.

Currently Implemented Features

The Emulator currently supports the following:

  • Lights API Get Lights/Update State, (1.1, 1.4, 1.5 and 1.6 from http://developers.meethue.com/1_lightsapi.html)
  • Schedules API Can create/update/delete Non-Recurring Schedules (Everything from http://developers.meethue.com/3_schedulesapi.html)
  • Groups API Can create Groups, run commands for default 0 group.
  • Adding new Bulbs Adding new bulbs is done via the File Menu Option.
  • 2 Graphical Views Hue bulbs are displayed in small view, or in a separate window frame showing their current colours/state.

TO DO

The Emulator is currently in development and will be updated as of when new features are implemented.

The following features will be added soon (roughly in the below order).

  • GUI Improvements Mouse over helpers for the Light Bulbs + scroll bars for when several lights are added.
  • Improve JUnit tests Currently tests are pretty basic with room for improvement.
  • Recurring Schedules Currently cannot create recurring schedules. Will be implemented soon.
  • Configurable Default Configuration Currently a default .json with 3 bulbs is loaded in. This will be changed to allow opening/saving external configs.
  • Light Effects/Looping/Transition Times Currently none of these features are implemented. The Bridge has a default transition time of 400 MS, emulator changes colours instantly currently.
  • UPNP Server Maybe a UPNP server so the emulator can be found during a Bridge Search. Undecided on this one.
